
# 业务规则:
1. 一个用户只能且必须关联一个角色。


# 后端规则
技术选型:
1. 项目使用 maven 管理依赖。
2. 项目使用 SpringBoot 2.7.18。
3. 不要使用 Lombok, 显示的定义get和set方法。
4. 项目中不要出现 Spring Bean 的循环依赖，不要使用 @Lazy 懒加载。

项目结构:
1. common 层为项目配置类、工具类、常量、枚举值等较通用的内容。
2. dao 层为项目与数据库交互的持久层，定义数据库表实体（entity文件夹下，实体以Entity结尾）和与实体交互的逻辑（以DAO结尾）。dao层禁止写复杂业务逻辑，dao 层内不能互相调用，也不能调用service层。
3. service 层为具体业务逻辑。
4. controller 层为项目暴露给外部的接口服务，该层中不应该存在太复杂的业务逻辑。


# 前端规则
前端编码规范:
1. 前端采用单页架构，导航栏和标签页栏常驻展示。
2. 前端页面风格保持一致。

前端代码结构:
1. common 文件夹存储全项目通用、无业务关联的公共代码。
2. modules 文件夹按菜单划分子文件夹存储业务代码，模块内仅定义私有样式，禁止编写全局样式，避免影响公共代码。模块内仅允许定义私有样式，严禁编写全局样式，防止污染公共样式环境。禁止重复实现公共代码已覆盖的功能逻辑。
