# 实验模板编辑器使用指南

## 一、支持的 Markdown 格式

实验模板编辑器支持标准 Markdown 语法，并扩展了自定义格式。

### 1. 标题

```markdown
# 一级标题
## 二级标题
### 三级标题
```

### 2. 文本格式

| 格式 | Markdown | 效果 |
|------|----------|------|
| 加粗 | `**加粗文本**` | **加粗文本** |
| 斜体 | `*斜体文本*` | *斜体文本* |
| 行内代码 | `` `代码` `` | `代码` |

### 3. 列表

无序列表：
```markdown
- 项目1
- 项目2
- 项目3
```

有序列表：
```markdown
1. 第一步
2. 第二步
3. 第三步
```

### 4. 表格

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 数据1 | 数据2 | 数据3 |
| 数据4 | 数据5 | 数据6 |
```

### 5. 链接

```markdown
[链接文本](https://example.com)
```

### 6. 代码块

````markdown
```javascript
function hello() {
  console.log('Hello World')
}
```
````

## 二、自定义 Markdown 格式

实验模板编辑器扩展了两种自定义格式，用于创建可交互的实验报告。

### 1. 填空项 `[placeholder]`

**用途**：创建需要学生填写内容的空白区域

**Markdown 语法**：
```markdown
请填写实验结论：*[实验结论]*
```

**HTML 输出**：
```html
<span class="inline-input" data-placeholder="实验结论">[实验结论]</span>
```

**显示效果**：渲染为带蓝色虚线边框的可填写区域

### 2. 公式 `$formula$`

**用途**：插入数学公式或特殊表达式

**Markdown 语法**：
```markdown
质能方程：$E=mc^2$
```

**HTML 输出**：
```html
<span class="inline-formula">$E=mc^2$</span>
```

**显示效果**：渲染为带橙色虚线边框的公式区域

## 三、插入组件

编辑器左侧工具栏提供了快捷插入组件功能。

### 1. 内联元素（点击插入）

#### 填空组件

- **操作**：点击「填空」按钮
- **效果**：在光标位置插入填空项 `[请输入]`
- **快捷方式**：直接输入 `[内容]` Markdown 语法

#### 公式组件

- **操作**：点击「公式」按钮
- **效果**：在光标位置插入公式 `$E=mc^2$`
- **快捷方式**：直接输入 `$公式$` Markdown 语法

### 2. 块级元素（拖拽插入）

#### 表格

- **操作**：将「表格」组件拖拽到画布
- **默认**：3行3列的表格
- **编辑**：
  - 点击单元格直接输入内容
  - 在右侧属性面板调整行数和列数

#### 分割线

- **操作**：将「分割线」组件拖拽到画布
- **效果**：插入水平分割线，用于内容分区

#### 图片

- **操作**：将「图片」组件拖拽到画布
- **配置**：
  - 方式一：在右侧属性面板输入图片 URL
  - 方式二：点击「选择图片」上传本地图片

## 五、模板示例

以下是一个完整的实验模板示例：

```markdown

1、观察甲醇燃烧的颜色变化。

2、测定甲醇燃烧的热值。

# 二、实验步骤

1、将甲醇倒入锅中。

2、点燃甲醇。

3、记录燃烧时间和颜色变化。

4、测量甲醇的质量。

# 三、实验数据记录与处理

1、观察到的颜色变化：*[甲醇燃烧颜色]*

2、燃烧热值计算：燃烧热值=（质量×燃烧时间×水的热容）÷燃烧温度升高。其中，水的热容为 $4.2J/g℃$。

| 质量 | 燃烧时间 | 燃烧升高温度 | 燃烧热值 |
|------|----------|--------------|----------|
|      |          |              |          |

# 四、实验结论

*[请填写实验结论]*
```

## 六、技术实现

### 依赖库

- **markdown-it**：Markdown 转 HTML
- **turndown**：HTML 转 Markdown

### 自定义规则位置

自定义 Markdown 规则定义在 `src/utils/markdown.js`：

```javascript
// 填空项规则
md.inline.ruler.before('emphasis', 'placeholder', function(state, silent) {
  // 匹配 [内容] 格式
})

// 公式规则
md.inline.ruler.before('emphasis', 'formula', function(state, silent) {
  // 匹配 $内容$ 格式
})
```

### 扩展新格式

如需添加新的自定义格式，在 `markdown.js` 中添加：

1. **解析规则**：使用 `md.inline.ruler.before()` 或 `md.block.ruler.before()`
2. **渲染规则**：在 `md.renderer.rules` 中定义渲染函数
3. **反向转换**：在 `turndownService.addRule()` 中定义 HTML → Markdown 规则

## 七、常见问题

### Q1: 为什么填空项显示为普通文本？

**A**: 检查是否使用了正确的语法 `[内容]`，确保方括号内不为空。

### Q2: 表格无法正确显示？

**A**: 确保表格格式正确，包含表头分隔行：

```markdown
| 列1 | 列2 |
|-----|-----|
| 数据 | 数据 |
```

### Q3: 如何修改已插入组件的属性？

**A**: 点击画布中的组件，右侧属性面板会显示可编辑的属性。

### Q4: 保存后内容丢失？

**A**: 确保点击了「保存模板」按钮，检查网络请求是否成功。

## 八、最佳实践

1. **模板命名**：使用清晰、描述性的名称，如"甲醇燃烧实验模板"
2. **结构清晰**：使用标题划分实验目的、步骤、数据、结论等部分
3. **合理填空**：在关键数据和结论处使用填空项
4. **表格设计**：预留足够的数据记录区域
5. **及时保存**：编辑过程中定期保存，避免数据丢失
